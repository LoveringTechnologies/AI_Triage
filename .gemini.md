# 🏥 AI Emergency Triage System - Project Overview

## 📌 Project Summary
A multi-platform clinical triage system utilizing a Random Forest ML model integrated with an advanced clinical logic layer. The system categorizes patients into a 5-zone clinical model and supports real-time staff notifications and manual clinical weighting.

## 🗂 File Structure
- `triage_dashboard.py`: Streamlit Web Dashboard (Nurse-facing).
- `triage_api.py`: FastAPI Backend (v1.4.1) with SQLite integration.
- `TriageMobile/App.js`: Fully featured React Native / Expo application.
- `triage.db`: SQLite database for users, triage logs, and notifications.
- `Model Training Data/`: High-fidelity training datasets.

## ⚖️ Clinical Weighting & Zone Engine (5-Zone Model)
The system maps AI predictions and clinical boosts to a 5-zone priority model:
1. **Zone 1 (Resuscitation)** - Red
2. **Zone 2 (Emergent)** - Orange
3. **Zone 3 (Urgent)** - Yellow
4. **Zone 4 (Less-Urgent)** - Blue
5. **Zone 5 (Non-Urgent)** - Green

### Intelligence Layers:
- **AI Layer**: Vitals-based priority prediction.
- **Location Layer**: Automatic boost for **Head/Spine** and **Critical Areas** (Chest, Neck).
- **Clinical Layer**: Staff can apply a **Manual Weight** (e.g., +1.0) to override AI zone assignment.

## 🛠 Setup & Execution

### Backend API
```bash
uvicorn triage_api:app --reload --host 0.0.0.0
```
*Note: Ensure `API_BASE` in `TriageMobile/App.js` matches your computer's current Local IP.*

### Mobile App
```bash
cd TriageMobile
npx expo start
```

## 🔔 Key Mobile Features
- **Responsive Drawer**: Slim staff portal (Left side).
- **Audit Logs**: Full database history viewing for logged-in staff.
- **Real-Time Alerts**: Automatic "Patient Alert" popups when new triages occur.
- **Zero-Start UI**: Dynamic DOB age calculation and persistent history visibility.
